## Структура проекта

В корне папки `team_4` находятся служебные файлы:
1. `.env` - для настройки токенов (необходимо настраивать самостоятельно)
2. `ontology.json` - файл с описанием онтологии по нашей теме
3. `requirements.txt` - зависимости проекта
4. `README.md` - файл, который Вы читаете

В папке `papers` хранятся статьи - то место, где происходит вся черновая работа по генерации статей. Только в самом конце её содержимое перенесётся в WEB

В папке `work` хранятся python-скрипты и jupyter-ноутбуки с экспериментами и работой

### Настройка

1. `py -3.10 -m venv .venv` - выбираем более стабильную версию
2. Для Windows: `.\.venv\Scripts\activate`
   
   Для Ubuntu: `source ./.venv/bin/activate`
3. `pip install WORK/team_4/requirements.txt`
4. `touch WORK/team_4/.env`

В файл `.env` поместите следующую информацию:
1. `AUTHORIZATION_KEY` - ключ авторизации для Gigachat API. Необходимо создать проект и создать новый ключ;
2. `SCOPE="GIGACHAT_API_PERS"` - обязательно для работы API;
3. `CA_BUNDLE_FILE` - путь к сертификатам минцифры, иногда требуется для корректной работы API.

### Визуализация графа

1. `python -m http.server 8000` - отрываем сервер на порту `8000` из папки `Visualization`
2. Переходим в `http://localhost:8000`

Сделано это для того, чтобы `CORS` не блокировали импорт графа в `JSON` формате.
