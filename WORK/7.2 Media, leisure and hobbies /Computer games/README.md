## Выполненные задачи

### ✅ 1. Изучение баз знаний
- Проанализированы WikiData, IGDB (Internet Game Database), MobyGames
- Составлены SPARQL-запросы для получения данных об играх, персонажах и разработчиках (см. `sparql_queries.txt`)
- Экспортированы данные в `raw_data/wikidata_games_export.json`
- Собрана информация по 35 ключевым понятиям из мира видеоигр

### ✅ 2. Построение онтологии
- Разработана концептуализация из 21 понятия по 7 разделам
- Построена диаграмма связей (см. `ontology.md`)
- Использованы связи:
  - **Иерархические** (Жанр → Поджанр: Стратегия → RTS, Пошаговая)
  - **Институциональные** (Разработчик → Издатель → Платформа)
  - **Атрибутивные** (Персонаж → Игра → Жанр)
  - **Документальные** (Статья → Раздел → Энциклопедия)

### ✅ 3. Генерация контента
- Настроено API для генерации текстов (аналог DashScope/Qwen)
- Сгенерировано 24 статьи через LLM (по 3 статьи на каждый раздел)
- **Промпт**: "Объясни для ребенка 10 лет, что такое [понятие] в мире компьютерных игр. Используй простые примеры из известных игр (Minecraft, Mario, GTA и др.). Добавь интересный факт."
- Примеры сгенерированных статей:
  - `articles/Technologies inside/Screen Magic.md`
  - `articles/How it all started/Tennis on TV.md`
  - `articles/Heroes and villains/Famous plumber.md`

### ✅ 4. Перекрёстные ссылки
- Реализован скрипт `src/link_concepts.py`
- Используется pymorphy2 для склонения названий игр и персонажей в русском языке
- Ссылки работают внутри раздела и между разделами
- Пример: в статье про Марио есть ссылки на `[[Nintendo]]`, `[[платформер]]`, `[[Боузер]]`

### ✅ 5. Визуализация и дизайн
- Подобраны изображения для каждой статьи (скриншоты игр, концепт-арты)
- Созданы иконки для 7 разделов (в папке `assets/icons/`)
- Разработан макет детской энциклопедии с ярким, понятным интерфейсом
