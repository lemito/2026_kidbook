# visualize_links — граф ссылок

Инструмент для визуализации внутренних и внешних ссылок внутри раздела `WEB/7.1_art/modern_technological_art`.

## Запуск

```bash
python3 tools/visualize_links.py
open tools/graph.html
```

Требований нет — только стандартная библиотека Python 3. Граф открывается в браузере как самодостаточный HTML-файл (без сервера).

## Что генерируется

Скрипт парсит `README.md` и все `articles/*.md`, собирает три типа связей и записывает `tools/graph.html`.

### Типы узлов

| Цвет | Тип | Описание |
|------|-----|----------|
| Янтарный | README | Главная страница раздела |
| По порталу | Статья | Файл из `articles/` |
| Серый | Wikipedia | Страница ru/en Wikipedia |
| Тёмно-серый | Сайт | Любой внешний домен |
| Красный | Вне репо | Ссылка на `.md`-файл из другого раздела вики |

### Типы рёбер

| Стиль | Тип |
|-------|-----|
| Сплошная синяя | Внутренняя ссылка (статья → статья) |
| Пунктир серый | Внешняя ссылка (статья → сайт/Wikipedia) |
| Пунктир красный | Ссылка вне репо |

## Макет

Граф строится в трёх концентрических кольцах от README:

```
        [П1]          ← портал 1, кольцо 1 (r≈260px)
    [П6]  ●  [П2]
        README
    [П5]  ●  [П3]
        [П4]          ← 6 секторов, по одному на портал

          ○ ○ ○       ← кольцо 3 (r≈740px): Wikipedia, сайты, вне-репо
```

- **Кольцо 0** (центр): README
- **Кольцо 1** (r≈260): статьи, напрямую связанные с README
- **Кольцо 2** (r≈490): статьи 2-го уровня (пока пусто — README ссылается на все статьи)
- **Кольцо 3** (r≈740): Wikipedia, внешние сайты, ссылки вне репо

Внутри каждого кольца статьи группируются по порталу (угловой сектор 60°).

## Управление в браузере

### Панель слева

**Поиск** — фильтрует узлы по названию статьи или домену. Совпадения подсвечиваются, остальное затемняется.

**Показать** — включить/выключить:
- Внутренние ссылки (по умолчанию вкл.)
- Внешние ссылки, Wikipedia (по умолчанию выкл.)
- Ссылки вне репо (по умолчанию выкл.)
- Подписи узлов

**Порталы** — скрыть/показать отдельные порталы.

**Путь A → B** — режим поиска кратчайшего пути:
1. Нажать «Включить режим пути»
2. Кликнуть узел A (подсветится зелёным)
3. Кликнуть узел B — путь отобразится зелёными рёбрами
4. Чекбокс «Через README» включает README как промежуточный узел (по умолчанию обходится, так как является хабом для всех статей)

### Взаимодействие с узлами

| Действие | Результат |
|----------|-----------|
| Клик на узел | Подсветить узел и его прямые связи; остальные рёбра скрыть |
| Клик на фон | Снять выделение |
| Наведение | Тултип с базовой информацией |
| Перетаскивание | Зафиксировать узел в новом месте |
| Прокрутка / пинч | Масштаб |

## Структура кода

```
visualize_links.py
├── Сбор узлов          — README + все articles/*.md
├── Парсинг ссылок       — LINK_RE, process_file()
│   ├── internal         — ссылки на .md внутри репо
│   ├── external         — http-ссылки (Wikipedia / сайты)
│   └── broken           — ссылки на .md вне репо
├── Статистика узлов     — in/out degree
└── HTML-шаблон          — D3.js v7, встроен как строка
    ├── computeRingDepths()   — BFS от README по внутренним рёбрам
    ├── computeTargets()      — целевые координаты (кольцо × сектор)
    ├── restart()             — пересборка D3 simulation
    ├── bfs()                 — поиск кратчайшего пути (A→B)
    └── recenter()            — центр канваса + пин README
```

## Добавление новых статей

Просто создать файл `articles/N.M_slug.md` с заголовком `# Название` — при следующем запуске скрипта статья автоматически появится в графе в нужном портале (номер определяется по префиксу `N.`).
