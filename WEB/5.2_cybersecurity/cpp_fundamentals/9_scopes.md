# 9. Область видимости и [время](../../1.2_natural_sciences/physics_in_everyday_life/Q20702.md) жизни переменных

![Картинка области видимости](covers/cover_9_scopes.png)

### 1. Великая тайна «Магических Границ»
<!--- Область видимости — это основа безопасности кода. Без неё переменные конфликтовали бы друг с другом на каждом шагу --->

Представь, что ты — главный герой в огромной ролевой игре ([RPG](../../7.2 Media, leisure and hobbies/Computer games/articles/heroes_and_villains/create_your_hero.md)). У тебя есть [инвентарь](../../6.1_Independent_living_and_daily_living_skills/Simple_and_safe_cooking/articles/how_to_read_recipe.md), в котором лежит **меч**. Пока ты находишься внутри «Подземелья Дракона», этот меч у тебя под рукой. Но как только ты выходишь из игры или переходишь в секретный [уровень](../../../8.1_entertainment/articles/gamification.md), где магия запрещена, твой меч может исчезнуть или стать невидимым.

В программировании всё точно так же! Переменные не живут вечно и не видны повсюду. У каждой переменной есть:
1.  **Область видимости (Scope)** — это «территория», на которой переменную знают и пускают «поиграть».
2.  **Время жизни (Lifetime)** — это время от момента «рождения» переменной до того момента, когда компьютер говорит: «Всё, ты мне больше не нужна», и стирает её из [памяти](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md).

### 2. [Блок](2_syntax.md) кода: Стены твоего дома
<!--- Блок кода определяется фигурными скобками {} и создает локальную область видимости --->

Прежде чем говорить о переменных, нужно понять, что такое **Блок кода**. 
Помнишь фигурные скобки `{ }` из уроков про функции или [циклы](../../1.2_natural_sciences/why_science_help_understand_world/patterns.md)? В [C](../../2.1_society/how_and_where_find_friends/articles/sora_drug.md)++ эти скобки — это не просто украшение. Это **магический забор**.

Все, что ты создаешь внутри одних скобок `{ }`, обычно «живет» только внутри них. Это как комната в твоем доме. Если ты положил свои носки в ящик в своей комнате, мама в гостиной их не увидит (даже если очень захочет).

```cpp
{
    int toy = 5; // Переменная "родилась" внутри скобок
    // Тут мы можем играть с toy
} 
// ОЙ! Тут скобка закрылась. Переменная toy "умерла". 
// Если ты попробуешь написать её имя здесь, компьютер скажет: "Кто это?"
```

### 3. Локальные переменные: Твои карманные [помощники](../../4.1_rules_of_study/how_to_learn_effectively/articles/digital_tools.md)
<!--- Локальные переменные создаются в стеке и уничтожаются при выходе из блока --->

**Локальные переменные** — это переменные, которые создаются внутри функции или блока кода. 

**Их [правила](../../2.1_society/cause_and_effect_relationships/articles/why_rules_work.md) жизни:**
*   **Где живут:** Только внутри своих `{ }`.
*   **Когда рождаются:** Когда компьютер доходит до строчки с их именем.
*   **Когда умирают:** Как только выполнение кода доходит до закрывающей скобки `}`.

**[Аналогия](../../1.2_natural_sciences/physics_in_everyday_life/Q46344.md):** Это как одноразовая тарелка на пикнике. Ты достал её (создал), положил в неё еду (присвоил [значение](../../7.2 Media, leisure and hobbies /useful_and_interesting_leisure/articles/leisure_and_why_need.md)), поел, и в конце пикника выбросил в мусорку. Тебе не нужно хранить её вечно.

```cpp
void dungeon() {
    int torch = 1; // Локальная переменная. Нужна только в этом подземелье.
} // Все, факел потух и исчез из памяти.
```

### 4. Глобальные переменные: [Звезды](../../1.2_natural_sciences/physics_in_everyday_life/Q1.md) мирового масштаба
<!--- Глобальные переменные доступны во всем файле, но их использование считается плохим тоном (anti-pattern) --->

Иногда нам нужно что-то, что будет видно **везде**. Например, имя игрока или уровень гравитации в игре. Такие переменные называются **Глобальными**.

Они объявляются в самом верху файла, вне всех функций.

**Их правила жизни:**
*   **Где живут:** По всему файлу. Любая функция может их видеть и менять.
*   **Когда рождаются:** В самом начале запуска программы.
*   **Когда умирают:** Только когда ты полностью закрываешь программу.

**Аналогия:** Это как **[Солнце](../../1.2_natural_sciences/physics_in_everyday_life/Q11388.md)**. Солнце видит и [человек](../../1.2_natural_sciences/physics_in_everyday_life/Q45003.md) в лесу, и человек в городе. Если кто-то (магическим образом) покрасит Солнце в зеленый [цвет](../../1.2_natural_sciences/physics_in_everyday_life/Q1075.md), его увидят зеленым все!

> [!CAUTION]
> **[Опасность](../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/06_ushib_kogda_vrach.md)!** Глобальные переменные — это очень коварная штука. Представь, что у тебя в программе 100 функций, и каждая может менять глобальную переменную `x`. Если `x` вдруг станет неправильным, тебе придется обыскать все 100 функций, чтобы найти «преступника». Поэтому программисты стараются использовать их как можно реже.

### 5. Статические переменные: Переменные с хорошей [памятью](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md)
<!--- Ключевое слово static изменяет время жизни переменной, сохраняя её значение между вызовами функции --->

Это самый необычный вид переменных. Слово `static` превращает обычную локальную переменную в «забывчивого робота» в «робота-мемуариста».

Обычно, когда функция заканчивает [работу](../../8.2_future/choosing_a_career_path/articles/interview.md), она забывает всё. Но статическая [переменная](3_variables.md) **помнит** своё значение, даже если функция «закрылась».

**Аналогия:** Представь копилку. Обычная переменная — это когда ты кладешь монетку, а когда уходишь из комнаты, [копилка](../../6.1_Independent_living_and_daily_living_skills/reasonable_spending/articles/savings.md) взрывается. Статическая переменная — это когда ты ушел, пришел через год, а монетка всё еще там!

```cpp
void clickCounter() {
    static int clicks = 0; // Статическая переменная. Она создается ОДИН раз.
    clicks++;
    std::cout << "Ты нажал кнопку " << clicks << " раз!" << std::endl;
}
```
Если вызвать эту функцию 3 раза, она выведет: 1, 2, 3. Если убрать слово `static`, она каждый раз будет выводить 1, 1, 1.

### 6. Время жизни (Lifetime): От рассвета до заката

Давай закрепим, сколько длится «[жизнь](../../1.2_natural_sciences/physics_in_everyday_life/Q1751973.md)» разных типов данных:

| [Тип](13_struct.md) переменной | Когда рождается? | Когда «умирает»? | Где её знают? |
|:---------------|:-----------------|:-----------------|:--------------|
| **Локальная**  | При создании в `{}` | На закрывающей `}` | Только внутри этих `{}` |
| **Глобальная** | При старте программы | При выходе из программы | Везде в этом файле |
| **Статическая**| При первом вызове  | При выходе из программы | Только в своей функции, но помнит всё! |

### 7. Почему [нельзя](../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/07_ushib_chego_nelzya.md) называть всех одинаково? (Конфликты имен)
<!--- Затенение (shadowing) происходит, когда локальная переменная имеет то же имя, что и глобальная --->

Представь, что в мире есть Глобальная переменная `int apple = 100;` (все яблоки в мире).
И внутри твоей функции ты создал Локальную переменную `int apple = 3;` (яблоки у тебя в кармане).

Какое яблоко выберет компьютер, если ты напишешь `cout << apple`?
**[Правило](../../1.2_natural_sciences/why_science_help_understand_world/patterns.md):** Компьютер всегда выбирает того, кто **ближе**. Локальное «яблоко» в кармане перекроет «мировое» яблоко. Это называется **Затенение**.

### 8. Практический пример: «[Легенда](../../7.2 Media, leisure and hobbies/Computer games/articles/game_culture/game_memes.md) о Счетчике»
Давай напишем [код](1_introduction.md), где участвуют все наши герои:

```cpp
#include <iostream>

// 1. Глобальная переменная (Солнце)
int globalWorldHealth = 100;

void healPlayer() {
    // 2. Локальная переменная (Аптечка)
    int potionPower = 20; 
    globalWorldHealth += potionPower;
    std::cout << "Мир стал чище! Здоровье мира: " << globalWorldHealth << std::endl;
}

void visitCastle() {
    // 3. Статическая переменная (Счетчик посещений)
    static int visitCount = 0; 
    visitCount++;
    std::cout << "Ты зашел в замок в " << visitCount << "-й раз." << std::endl;
}

int main() {
    std::cout << "Начало игры. Здоровье мира: " << globalWorldHealth << std::endl;

    healPlayer();
    healPlayer();

    visitCastle();
    visitCastle();
    visitCastle();

    // Тут нельзя написать: cout << potionPower; 
    // Потому что potionPower "умерла" внутри функции healPlayer.

    return 0;
}
```

### 9. Главные [советы](../../7.2 Media, leisure and hobbies /useful_and_interesting_leisure/articles/mistakes_in_choosing_hobby.md) для начинающего архитектора

1.  **Держи переменные взаперти.** Старайся делать переменные локальными. Чем меньше «территория», где видна переменная, тем меньше шансов, что ты случайно её испортишь.
2.  **Глобальные — только для констант.** Если тебе нужно что-то глобальное, лучше сделай это неизменяемым (используй `const`). Например, `const double PI = 3.14;`.
3.  **Не делай [блоки](../../1.2_natural_sciences/physics_in_everyday_life/Q169019.md) слишком глубокими.** Если у тебя скобки внутри скобок, которые внутри других скобок... (как матрешка), в них очень легко запутаться!
4.  **Статик — это для памяти.** Используй статические переменные только тогда, когда функции действительно нужно что-то [помнить](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md) между вызовами.
