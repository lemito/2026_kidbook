# 14. Динамическая [память](../../3.1. healthy lifestyle/Sleep, nutrition, and adolescent energy/articles/sleep_and_memory_grades.md): Магия расширения твоего рюкзака

![Картинка динамическая память](covers/cover_14_dynamic_memory.png)

Привет, мастер-программист! Мы уже много знаем о переменных и массивах. Но у них есть одна «проблема»: они как обычный школьный рюкзак. Если ты купил рюкзак на 5 учебников, ты не сможешь засунуть туда 10. А если положишь всего один — остальное место будет гулять зря.

В [C](../../2.1_society/how_and_where_find_friends/articles/sora_drug.md)++ есть суперспособность — **Динамическая память**. Это как магический рюкзак, который может увеличиваться или уменьшаться прямо на ходу, пока работает [программа](../../5.1_technology_and_digital_literacy/operating system/articles/process.md)!

Давай узнаем, как пользоваться этой магией.

---

## 1. Стеки и Кучи: Где лежат твои игрушки?

В компьютере есть два главных места для хранения данных:

1.  **[Стек](../../5.1_technology_and_digital_literacy/operating system/articles/process.md) (Stack):** Это как маленькая полочка рядом с рабочим столом. Всё, что ты там хранишь (обычные переменные `int x`, `double y`), создается само и само исчезает, когда ты заканчиваешь [работу](../../8.2_future/choosing_a_career_path/articles/interview.md). Но места там — всего чуть-чуть.
2.  **[Куча](../../5.1_technology_and_digital_literacy/operating system/articles/process.md) (Heap):** Это огромный склад за городом. Там места — завались! Но есть нюанс: чтобы положить туда что-то или забрать, тебе нужно написать специальное «заявление» и самому следить за порядком.

---

## 2. [Оператор](../../3.2 healthy lifestyle/how to act in a dangerous situation/articles/emergency-112.md) `new` — Заказываем место на складе

Чтобы получить кусочек [памяти](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md) в «Куче», нам нужно слово `new`. Но поскольку на складе нет имен (как `x` или `y`), нам выдают только **[адрес](../../5.1_technology_and_digital_literacy/how_internet_works/articles/ip_mac/ip_and_mac.md)** (номер ячейки). А где мы храним адреса? Правильно, в **указателях**!

```cpp
int* p = new int; // Мы просим: "Дайте мне одну ячейку для целого числа"
```

Теперь у нас есть указатель `p`, который «смотрит» на новую свободную комнатку на складе. Мы можем положить туда число:

```cpp
*p = 42; 
cout << "В магической ячейке лежит: " << *p;
```

---

## 3. Оператор `delete` — Уборка за собой

Это **самое важное [правило](../../1.2_natural_sciences/why_science_help_understand_world/patterns.md)** динамической памяти. Если ты взял поиграть игрушку со склада, ты **обязан** её вернуть. Если ты этого не сделаешь, место на складе будет считаться занятым вечно.

Для этого используется [команда](../../4.1_rules_of_study/how_to_learn_effectively/articles/peer_learning.md) `delete`:

```cpp
delete p; // Мы говорим: "Спасибо, эта комната мне больше не нужна!"
```

После этой команды указатель `p` всё еще существует, но он больше не имеет права заглядывать в ту комнату. Это как если бы ты вернул ключи от номера в отеле.

---

## 4. Динамические массивы: Рюкзак любого размера

Обычно мы создаем массивы так: `int arr[10];`. Но что если мы не знаем, сколько чисел введет пользователь? Может 5, а может 5000?

Тут на [помощь](../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/10_krovotechenie_chto_delat.md) приходят динамические массивы:

```cpp
int size;
cout << "Сколько элементов тебе нужно? ";
cin >> size;

// Создаем массив именно того размера, который ввел пользователь!
int* mas = new int[size]; 

// Теперь можно работать с ним как с обычным массивом
mas[0] = 100;
mas[1] = 200;

// И самое важное — в конце удаляем его!
// Для массивов пишем со скобками: delete[]
delete[] mas; 
```

---

## 5. Монстр по имени «Утечка памяти»

Если ты будешь постоянно использовать `new`, но забывать писать `delete`, случится страшное — **Утечка памяти (Memory Leak)**.

Представь: ты каждый день заказываешь пиццу, съедаешь её, но пустые коробки не выбрасываешь, а оставляешь в комнате. Сначала всё будет хорошо. Потом станет тесно. А через месяц ты просто не сможешь зайти в комнату!

Так и с компьютером: если программа «ест» память и не возвращает её, компьютер начнет тормозить, а потом программа просто «вылетит» с ошибкой.

---

## 6. Как не запутаться? (Три золотых [правила](../../2.1_society/cause_and_effect_relationships/articles/why_rules_work.md))

Чтобы быть крутым программистом и не ломать компьютер, запомни:

1.  **На каждый `new` должен быть свой `delete`.** Это как пара ботинок — левый и правый.
2.  **На каждый `new[]` должен быть свой `delete[]`.**
3.  **Не используй `delete` дважды для одного и того же адреса.** Это всё равно что пытаться вернуть в библиотеку книгу, которую ты уже вернул.

---

## 7. Пример: Магический [список](10_arrays.md) друзей

Давай напишем кусочек кода, который спрашивает, сколько у тебя друзей, и создает список их имен:

```cpp
int count;
cout << "Сколько у тебя друзей? ";
cin >> count;

// Выделяем память под массив имен
string* friends = new string[count];

for(int i = 0; i < count; i++) {
    cout << "Введите имя друга №" << i + 1 << ": ";
    cin >> friends[i];
}

cout << "Твой список готов!" << endl;

// ... работаем со списком ...

// Убираем за собой!
delete[] friends;
```

## 8. А можно попроще?

На самом деле, современные программисты на C++ стараются не использовать `new` и `delete` вручную слишком часто. Для этого придумали умные штуки, такие как **векторы** (`std::vector`). 

[Вектор](../../1.2_natural_sciences/physics_in_everyday_life/Q161635.md) — это как автоматический робот-рюкзак, который сам делает `new`, когда нужно расшириться, и сам делает `delete`, когда ты закончишь работу. О них мы поговорим в следующей статье!

## Итог урока

1.  **Динамическая память** — это способ брать столько места, сколько нужно прямо сейчас.
2.  **`new`** — это команда «дай мне место».
3.  **`delete`** — это команда «я закончил, забирай обратно».
4.  **Утечка памяти** — это когда мы забыли сделать `delete`.
5.  Всегда следи за своими указателями!

Ты молодец! Динамическая память — это сложная тема даже для взрослых, но ты с ней справился. Теперь ты умеешь управлять [памятью](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md) компьютера как настоящий профессионал!
