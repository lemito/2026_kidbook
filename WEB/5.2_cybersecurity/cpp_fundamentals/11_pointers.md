# Указатели и ссылки: как найти сокровище в [памяти](../../4.1_rules_of_study/how_to_memorize/articles/pamyat.md) компьютера

![Карта сокровищ с адресом в памяти](covers/cover_11_pointers.png)

Представь, что компьютерная [память](../../3.1. healthy lifestyle/Sleep, nutrition, and adolescent energy/articles/sleep_and_memory_grades.md) — это огромный [город](../../3.2 healthy lifestyle/how to act in a dangerous situation/articles/lost-in-city.md) с тысячами домов. В каждом доме живет какая-то [информация](../../5.1_technology_and_digital_literacy/information and media literacy/как_устроена_современная_информационная_среда.md) (число, буква или целое предложение). У каждого дома есть свой уникальный номер — **[адрес](../../5.1_technology_and_digital_literacy/how_internet_works/articles/ip_mac/ip_and_mac.md)**.

До этого момента мы просто давали вещам имена: «В этой коробке под названием `apple` лежит число 5». Но профессиональные программисты умеют работать не только с самими «коробками», но и с их адресами. 

В [C](../../2.1_society/how_and_where_find_friends/articles/sora_drug.md)++ для этого есть два супер-инструмента: **Указатели** и **Ссылки**. Давай разберемся, как ими пользоваться и не запутаться! 🗺️

---

## 1. Что такое [адрес памяти](../../5.1_technology_and_digital_literacy/operating system/articles/physical_memory.md)?

Когда ты создаешь переменную, компьютер выделяет ей «домик» в оперативной памяти. 
Чтобы узнать адрес этого домика, в C++ используется символ амперсанда — `&`.

**[Аналогия](../../1.2_natural_sciences/physics_in_everyday_life/Q46344.md):** [Переменная](3_variables.md) — это твой друг. А `&переменная` — это его домашний адрес, записанный на бумажке.

```c++
#include <iostream>

int main() {
    int cookies = 10; // Сама переменная

    std::cout << "Количество печенья: " << cookies << std::endl;
    // Выводим адрес, где это печенье лежит в памяти
    std::cout << "Адрес печенья в памяти: " << &cookies << std::endl;

    return 0;
}
```
*В консоли ты увидишь что-то странное, вроде `0x7ffeb3`, — это и есть номер «дома» в памяти в шестнадцатеричном виде.*

## 2. Указатели: «Записка с адресом»

**Указатель** — это специальная переменная, которая хранит в себе не само число, а **адрес** другой переменной.

Чтобы создать указатель, мы добавляем звездочку `*` к типу данных.

```c++
int main() {
    int magicNumber = 42;
    
    // Создаем указатель p. Он "смотрит" на целые числа (int*)
    // Кладем в него адрес нашего магического числа
    int* p = &magicNumber;

    std::cout << "Адрес в указателе: " << p << std::endl;
    return 0;
}
```

> [!NOTE]
> Звездочка `*` в C++ хитрая. Когда она стоит при создании переменной (`int* p`), она значит «это указатель». А когда она стоит перед уже готовым указателем (`*p`), она делает совсем другое!

## 3. Разыменование: «Идем в гости»

Если у тебя есть записка с адресом (указатель), ты можешь пойти по этому адресу и посмотреть, что там лежит, или даже поменять там всё! Это называется **разыменование**. Для этого снова используется `*`.

**Аналогия:** У тебя есть [карта](../../5.1_technology_and_digital_literacy/information and media literacy/карта_компетенций_по_возрастам.md) сокровищ. Указатель — это сама карта. Разыменование — это когда ты пришел по карте к крестику и начал копать клад.

```c++
int main() {
    int chest = 100; // В сундуке 100 золотых
    int* map = &chest; // Карта указывает на сундук

    // Смотрим через карту, что в сундуке
    std::cout << "В сундуке: " << *map << " золотых." << std::endl;

    // Меняем содержимое через карту!
    *map = 0; // Кто-то украл золото!
    
    std::cout << "Теперь в сундуке: " << chest << " золотых." << std::endl;
    return 0;
}
```

## 4. Ссылки: «Второе имя»

**[Ссылка](../../4.2_thinking_and_working_information/how_to_search_information/articles/copypaste.md)** — это гораздо проще. Это просто псевдоним (никнейм) для уже существующей переменной. У ссылки нет своего адреса, она — то же самое, что и оригинал, просто под другим именем.

Чтобы создать ссылку, используй `&` после типа данных.

**Аналогия:** Тебя зовут Александр, но [друзья](../../4.1_rules_of_study/how_to_learn_effectively/articles/peer_learning.md) зовут тебя Саня. Ты один и тот же [человек](../../1.2_natural_sciences/physics_in_everyday_life/Q45003.md), просто имен два. Если Саня съест конфету, то и Александр станет сытым.

```c++
int main() {
    int alex = 5;
    int& sanya = alex; // sanya — это ссылка на alex

    sanya = 10; // Меняем "Саню"
    
    std::cout << "Алекс теперь: " << alex << std::endl; // Тоже 10!
    return 0;
}
```

> [!IMPORTANT]
> Ссылку [нельзя](../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/07_ushib_chego_nelzya.md) оставить «пустой». Она должна сразу при создании за что-то «схватиться». А указатель может быть пустым и указывать в никуда.

---

## [Сравнение](5_operators.md): Указатели vs Ссылки

| Свойство | Указатель (`*`) | Ссылка (`&`) |
| :--- | :--- | :--- |
| **Что это?** | Переменная с адресом | Второе имя (псевдоним) |
| **Можно ли оставить пустым?** | Да (nullptr) | Нет, никогда |
| **Можно ли переключить?** | Да, может указывать на другой дом | Нет, приклеена навсегда |
| **Сложность** | Сложно (нужны `*` и `&`) | Просто (как обычная переменная) |

## Зачем это нужно?

Ты можешь спросить: «Зачем мне эти сложности с адресами, если можно просто пользоваться именами?». Есть две главные причины:

1. **[Скорость](../../1.2_natural_sciences/physics_in_everyday_life/Q11402.md):** Если у тебя в программе есть огромный сундук с данными (например, карта всей игры), передавать его из одной части программы в другую очень долго — компьютер будет долго его копировать. Проще передать маленькую «записку с адресом» (указатель).
2. **Изменение данных:** Если ты хочешь, чтобы функция ([команда](../../4.1_rules_of_study/how_to_learn_effectively/articles/peer_learning.md)) поменяла твою переменную, ей нужно дать доступ к «дому» этой переменной через ссылку или указатель.

---

## [Опасности](../../1.2_natural_sciences/physics_in_everyday_life/Q845744.md): Указатель «в никуда»

Если ты создашь указатель и не дашь ему адрес, он будет указывать на случайное место в памяти. Это как если бы на карте сокровищ был адрес... кабинета директора школы или полицейского участка. Попытка «разыменовать» такой указатель приведет к тому, что [программа](../../5.1_technology_and_digital_literacy/operating system/articles/process.md) сломается.

> [!WARNING]
> Всегда инициализируй указатели! Если адреса пока нет, пиши `nullptr`. Это специальное слово, которое значит «указатель в никуда».

```c++
int* myPointer = nullptr; // Безопасно
```

## Подведём итоги

Указатели и ссылки — это мощные [инструменты](../../1.2_natural_sciences/physics_in_everyday_life/Q36253.md), которые делают тебя «мастером памяти». 
- **Адрес (`&`)** — это где лежит переменная.
- **Указатель (`*`)** — это переменная-записка с адресом.
- **Ссылка (`&` в типе)** — это просто другое имя.

Сначала это может казаться запутанным, но как только ты привыкнешь к «картам сокровищ», твой код станет гораздо быстрее и профессиональнее!
