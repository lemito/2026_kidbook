# Что такое [сервер](../http_https/http_https.md) и где он находится

Каждый раз, когда ты открываешь сайт, смотришь [видео](../../../information and media literacy/оценка_качества_изображений_и_видео.md) на YouTube или играешь в онлайн-игру, где-то в мире работает **сервер** — он слышит твой [запрос](../http_https/http_https.md) и отвечает на него. Что это за [устройство](../../../../1.2_natural_sciences/physics_in_everyday_life/Q178032.md)? Где оно стоит? Как оно справляется с миллионами запросов одновременно?

![Серверная стойка в дата-центре — ряды блоков с мигающими лампочками](../../images/3.png)

## Сервер — это просто компьютер?

По сути — да. **Сервер** — это компьютер, который **обслуживает запросы** других компьютеров. Слово «сервер» происходит от английского *server* — «тот, кто обслуживает, официант».

Но есть важные отличия от обычного домашнего компьютера:

| Обычный компьютер | Сервер |
|-------------------|--------|
| Используется одним человеком | Обслуживает тысячи запросов одновременно |
| Включают и выключают по необходимости | Работает **24/7** без остановок |
| Есть монитор, [клавиатура](../../../../7.1_art/musical_instruments/articles/piano.md) | Обычно **нет монитора** — управление по сети |
| Важна [скорость](../../../../1.2_natural_sciences/physics_in_everyday_life/Q11402.md) для игр/видео | Важна **надёжность и стабильность** |
| Обычный жёсткий [диск](../../../operating system/articles/file_system.md) | Множество жёстких дисков с **резервированием** |
| Один [блок](../../../../5.2_cybersecurity/cpp_fundamentals/2_syntax.md) питания | Два и более блоков питания (если один сломается — работает второй) |

> **Знаешь ли ты?** Слово «сервер» имеет два значения. **Сервер-железо** — это физическое устройство. **Сервер-программа** — это программное обеспечение, которое принимает запросы и отвечает на них. Например, Nginx — это программа-сервер, которая может работать даже на твоём ноутбуке.

## Что делает веб-сервер

Когда ты открываешь сайт, [браузер](../http_https/http_https.md) отправляет [HTTP-запрос](../http_https/http_https.md) на **веб-сервер**. Задача веб-сервера:

1. **Принять** запрос от браузера
2. **Разобраться**, что именно просят (`GET /index.html`)
3. **Найти или сформировать** нужные [данные](../../../../2.1_society/cause_and_effect_relationships/articles/ai_causality.md)
4. **Отправить [ответ](../http_https/http_https.md)** обратно браузеру

Звучит просто. Но хороший веб-сервер делает это для **тысяч пользователей одновременно**, не теряя ни одного запроса.

### Статические и динамические сайты

**Статический сайт** — это набор готовых файлов: [HTML](../../../../7.1_art/modern_technological_art/articles/2.1_jodi.md), CSS, картинки. Сервер просто находит [файл](../../../operating system/articles/file_system.md) и отдаёт его. Как библиотека: пришёл, попросил книгу, тебе её дали.

**Динамический сайт** — сервер **формирует страницу** под конкретного пользователя в момент запроса. Страница твоего профиля ВКонтакте или [результаты](../../../../1.2_natural_sciences/why_science_help_understand_world/research_work.md) поиска Google — всегда разные, они генерируются на лету.

```
Статический запрос:
Браузер → "Дай мне /about.html" → Сервер ищет файл → Отдаёт файл

Динамический запрос:
Браузер → "Покажи профиль пользователя 12345"
→ Сервер запрашивает данные из базы данных
→ Подставляет данные в шаблон
→ Генерирует HTML прямо сейчас
→ Отдаёт браузеру
```

## Программы-веб-серверы

На физическом сервере работает [программа](../../../operating system/articles/process.md), которая обрабатывает HTTP-запросы. Самые популярные:

| Программа | Кто создал | Особенности |
|-----------|-----------|-------------|
| **Nginx** | Игорь Сысоев (Россия, 2004) | Очень быстрый, эффективный, популярный |
| **Apache** | Apache Foundation ([1995](../../../../7.1_art/modern_technological_art/articles/2.5_siberian_deal.md)) | Старейший, очень гибкий |
| **Caddy** | Matt Holt (2015) | Автоматически настраивает [HTTPS](../http_https/http_https.md) |
| **IIS** | Microsoft | Для серверов на [Windows](../../../operating system/articles/operating_system.md) |

Сегодня больше половины всех сайтов в мире работают под управлением **Nginx** или **Apache**.

> **Знаешь ли ты?** Nginx придумал и написал российский [программист](../../../../8.2_future/choosing_a_career_path/articles/programmer.md) Игорь Сысоев. В 2002 году он создавал его для сайта Rambler.ru, которому нужно было обрабатывать [500](../http_https/http_https.md) миллионов HTTP-запросов в сутки. Сегодня Nginx используют Netflix, Airbnb, GitHub, Instagram и тысячи других сервисов.

## [Виды](../../../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/08_porezy_sadiny_vidy.md) серверов

Не все [серверы](../../../operating system/articles/operating_system.md) — веб-серверы. Разные серверы выполняют разные [задачи](../../../../1.2_natural_sciences/why_science_help_understand_world/research_work.md):

| [Тип](../../../../5.2_cybersecurity/cpp_fundamentals/13_struct.md) сервера | Что делает | Пример |
|-------------|-----------|--------|
| **Веб-сервер** | Отдаёт HTML-страницы браузеру | Nginx, Apache |
| **Сервер [базы данных](../../../../7.1_art/modern_technological_art/articles/2.2_heath_bunting.md)** | Хранит и выдаёт данные | MySQL, PostgreSQL |
| **Файловый сервер** | Хранит и раздаёт файлы | FTP-сервер |
| **Почтовый сервер** | Принимает и отправляет email | Gmail-серверы Google |
| **DNS-сервер** | Переводит имена в IP-адреса | [DNS](../dns/dns.md) <!-- TODO: статья пишется --> |
| **[Игровой сервер](../tcp_udp/online_games.md)** | Синхронизирует состояние игры | Серверы [Minecraft](../tcp_udp/online_games.md) |
| **Стриминговый сервер** | Раздаёт видео по частям | YouTube, Netflix |

Когда ты смотришь видео на YouTube, браузер общается сразу с несколькими серверами: один отдаёт HTML-страницу, другой — само видео, третий — [комментарии](../../../../4.2_thinking_and_working_information/how_to_search_information/articles/cooperative_work.md), четвёртый — рекламу.

## Где физически находятся серверы

### Дата-центр: дом для серверов

Серверы стоят в специальных зданиях — **дата-центрах** (Data Center, ЦОД — Центр Обработки Данных).

<!-- Картинка: вид сверху или снаружи крупного дата-центра — большое промышленное 
9.png
здание, много кондиционеров на крыше -->

Дата-центр — это не просто склад с компьютерами. Это сложная инженерная система:

**[Питание](../../../../3.1. healthy lifestyle/Sleep, nutrition, and adolescent energy/articles/breakfast_for_the_brain.md):**
- Несколько независимых линий электропитания от разных подстанций
- Мощные [генераторы](../../../../1.2_natural_sciences/physics_in_everyday_life/Q988780.md) на дизельном топливе — на случай отключения света
- **UPS** ([источники](../../../../4.2_thinking_and_working_information/how_to_search_information/articles/three_whales.md) бесперебойного питания) — дают [время](../../../../1.2_natural_sciences/physics_in_everyday_life/Q20702.md) запустить генераторы

**[Охлаждение](../../../../1.2_natural_sciences/physics_in_everyday_life/Q182453.md):**
- Серверы выделяют огромное количество тепла
- Мощные системы кондиционирования работают круглосуточно
- Некоторые дата-центры строят рядом с реками или озёрами для охлаждения водой
- **Google** экспериментирует с дата-центрами прямо на морских платформах

**[Безопасность](../../../../1.2_natural_sciences/neurobiology_for_teens/articles/17_hugs_oxytocin.md):**
- Пропускной [режим](../../../../4.1_rules_of_study/how_to_learn_effectively/articles/breaks_and_rest.md), охрана, видеонаблюдение
- Биометрические считыватели на входе
- Серверы физически не доступны посторонним
- Дублирование всего: два интернет-провайдера, два блока питания, два кондиционера

**[Связь](../../../../1.2_natural_sciences/physics_in_everyday_life/Q12969754.md):**
- Несколько независимых оптоволоконных кабелей
- [Подключение](../../../../1.2_natural_sciences/physics_in_everyday_life/Q25250.md) к **IX** (Internet Exchange) — точкам обмена трафиком, где пересекаются сети разных провайдеров

### Серверные стойки

Внутри дата-центра серверы не стоят просто на полках. Их устанавливают в специальные **серверные стойки** (racks) высотой 2 метра. Размер серверов измеряется в **юнитах** (U): 1U = 4.45 см. В одну стойку помещается до 42U — то есть 42 сервера высотой 1U.

```
┌──────────────┐
│  Сервер 1U   │  ← Например, веб-сервер
├──────────────┤
│  Сервер 1U   │  ← База данных
├──────────────┤
│  Сервер 2U   │  ← Сервер хранения данных
├──────────────┤
│  Коммутатор  │  ← Сетевое оборудование
│     1U       │
└──────────────┘
```

## [CDN](../dns/cdn.md) — серверы везде

Представь: пользователь из Владивостока открывает сайт, серверы которого стоят в Москве. Данные должны пройти 9 000 км. [Скорость света](../../../../1.2_natural_sciences/physics_in_everyday_life/Q12969754.md) в оптоволокне — около [200](../http_https/http_https.md) 000 км/с. Только физически этот [путь](../../../../1.2_natural_sciences/physics_in_everyday_life/Q11476.md) занимает ~45 мс в одну сторону и 90 мс туда-обратно. А ещё есть маршрутизаторы, задержки... В итоге сайт будет открываться медленно.

[Решение](../../../../2.1_society/cause_and_effect_relationships/articles/personal_choice.md) — **CDN** (Content Delivery Network, [сеть доставки контента](../dns/cdn.md)). CDN — это сотни серверов, расставленных по всему миру. Когда ты запрашиваешь файл, CDN отдаёт его с **ближайшего** к тебе сервера.

<!-- Картинка: карта мира с точками CDN-серверов, расставленными по разным городам, и стрелками к пользователям -->

| Без CDN | С CDN |
|---------|-------|
| Один сервер, один регион | Сотни серверов по всему миру |
| Долго для удалённых пользователей | Быстро для всех |
| Легко «завалить» трафиком | Нагрузка распределена |
| Один точка отказа | [Отказ](../../../../2.1_society/how_and_where_find_friends/articles/otkaz_ne_konets.md) одного узла незаметен |

Крупнейшие CDN-провайдеры: **[Cloudflare](../dns/cdn.md)**, **[Akamai](../dns/cdn.md)**, **Fastly**. Они обрабатывают огромную долю всего интернет-трафика в мире.

> **Знаешь ли ты?** Когда YouTube начинал в 2005 году, серверы стояли в гараже Чада Хёрли — одного из основателей. За несколько месяцев [трафик](../dns/cdn.md) вырос настолько, что им пришлось срочно переезжать в настоящий дата-центр. Сейчас YouTube каждую минуту загружают более 500 часов нового видео.

## Облачные серверы

Раньше компании покупали или арендовали физические серверы. Сегодня большинство используют **облачные [сервисы](../../../../4.1_rules_of_study/how_to_learn_effectively/articles/digital_tools.md)**: AWS (Amazon), Google Cloud, Microsoft Azure, Yandex Cloud.

В облаке ты не берёшь в аренду конкретный физический сервер — ты берёшь **вычислительную [мощность](../../../../1.2_natural_sciences/physics_in_everyday_life/Q25236.md)**. Можно включить 1000 серверов на час и выключить — заплатишь только за этот час. Это называется **эластичность**: [ресурсы](../../../../2.1_society/cause_and_effect_relationships/articles/ecological_footprint.md) масштабируются под текущую нагрузку.

Например, магазин может запустить дополнительные серверы в Чёрную пятницу, когда покупателей в 10 раз больше обычного, а потом вернуть их.

## Отказоустойчивость — как сервер не ломается

Крупные сервисы (Google, Facebook, YouTube) не могут позволить себе «упасть» даже на минуту — это миллионы рублей убытков и миллионы недовольных пользователей.

Для защиты от сбоев используют:

**Репликация** — все данные хранятся в нескольких копиях на разных серверах. Если один сломался — данные не теряются.

**Load Balancing (балансировка нагрузки)** — специальная программа распределяет запросы между несколькими серверами. Ни один не перегружается.

**Резервные дата-центры** — копия всей инфраструктуры в другом городе или стране. Если дата-центр в Москве отключат — заработает резерв в Петербурге.

**Мониторинг** — системы следят за каждым сервером 24/7 и мгновенно сообщают о проблемах.

> **Знаешь ли ты?** У Amazon Web Services были громкие аварии, когда из-за одной [ошибки](../../../../3.1_healthy_lifestyle/pervaya_pomoshch/ushibi_porezy_ozhogi/07_ushib_chego_nelzya.md) инженера падали сотни тысяч сайтов и приложений по всему миру. Даже самые надёжные системы иногда ломаются — именно поэтому резервирование так важно.

## Интересные [факты](../../../../1.2_natural_sciences/physics_in_everyday_life/Q17737.md)

- **Google** потребляет около **15 миллиардов** киловатт-часов электроэнергии в год — столько же, сколько небольшая страна.
- **Facebook** (Meta) хранит более **100 петабайт** фотографий. 1 петабайт = 1 000 000 гигабайт.
- **Подводные кабели** длиной десятки тысяч километров соединяют дата-центры на разных континентах — по ним идёт весь межконтинентальный интернет-трафик.
- **Microsoft** тестировала подводный дата-центр — [контейнер](../../../../5.2_cybersecurity/cpp_fundamentals/15_stl.md) с серверами на дне Северного моря у берегов Шотландии. Морская [вода](../../../../3.1. healthy lifestyle/Sleep, nutrition, and adolescent energy/articles/drinking_regime.md) идеально охлаждает серверы.
- **Первый сервер** (компьютер NeXT) до сих пор хранится в ЦЕРНе. На нём написано от руки: «This machine is a server. DO NOT POWER [IT](../../../../8.2_future/choosing_a_career_path/articles/programmer.md) DOWN!!» — «Это сервер. НЕ ВЫКЛЮЧАТЬ!!»
